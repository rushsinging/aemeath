# Bug #38: Assistant 空消息导致 API 400 invalid_request_error

**状态**：已修复并确认（2026-05-18）
**优先级**：中
**发现日期**：2026-05

## 症状

会话调用模型时返回：

```text
Error: API error [400 Bad Request]: {"error":{"message":"Invalid assistant message: content or tool_calls must be set","type":"invalid_request_error","param":null,"code":"invalid_request_error"}}
```

包含该异常 assistant 消息的 session 后续无法继续正常请求 OpenAI-compatible API；resume 或继续对话会反复触发 400。

## 根因

assistant 消息仅包含 `reasoning_content`（思考内容），没有可见的 `content` 和 `tool_calls`。原过滤条件要求三者都为空才跳过，导致 reasoning-only 的 assistant 消息被发送到 API，违反校验要求。

## 修复方案

在 `message_conversion.rs` 中放宽 assistant 消息的过滤条件：只要 `content_parts` 和 `tool_calls` 都为空即跳过，不再要求 `reasoning_content` 也为空。这样 reasoning-only 的 assistant 消息会在发送前被丢弃。

## 验证

- 新增回归测试 `test_convert_messages_drops_reasoning_only_assistant` 覆盖只包含 Thinking 块的 assistant 消息被过滤。
