# Bug #30: 对话过程中 input queue 不被消费，消息卡在队列里

**优先级**：高
**状态**：已修复
**修复 commit**：`1287be5`
**发现日期**：2026-05
**归档日期**：2026-05-09

## 症状

对话进行中（is_processing=true）用户在 input area 排队多条消息（已有 input queue 显示新条目），但 agent loop 完成当前 turn 后，queue 中的内容没有被弹出消费，消息一直滞留在 TUI input queue 区域，下一轮 LLM 完全看不到。

## 复现路径

1. 用户发出第一条消息，agent 进入处理（开始 streaming / tool call）
2. 处理期间用户在 input area 输入并 Enter 提交多条消息
3. queue 区域显示这些消息已排队
4. 第一条消息处理完成（agent stop_reason=EndTurn，tool call 全跑完）
5. **预期**：agent loop 自动 pop queue，把第一条排队消息作为新 user turn 输入
6. **实际**：agent 完全停下，queue 内容仍显示在 TUI 上不被消费

## 根因

1. **outer loop 的 pop 路径断了**：Feature #7 设计的"user-turn 末尾 pop queue"逻辑在某次重构后失效
2. **Feature #12 双层循环未实施完成**：A/B 检查点未接入
3. **状态机转换问题**：`is_processing` 退回 false 时未触发 queue check
4. **ask_user / 中断态遮蔽**：之前 Bug #19 修过 ask_user 等待时 queue 不消费的问题，可能类似遮蔽逻辑误判常规处理结束态
