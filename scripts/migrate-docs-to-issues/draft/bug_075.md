# Bug #75：中文输入法下 input area 输入顺序错乱（查看 → 看查）

| 字段 | 值 |
|------|-----|
| 优先级 | 中 |
| 发现日期 | 2026-05 |
| 归档日期 | 2026-05-30 |
| 状态 | 已确认修复 |
| 根因类别 | TUI 输入数据流方向 / CJK 字节-字符索引 |

## 症状

启用中文输入法（IME）时，在 TUI input area 输入"查看"后，input 中实际显示为"看查"，字符顺序被颠倒。

## 根因

旧架构下 `mirror_input_area_to_model`（已删除的 `apps/cli/src/tui/core/input_bridge.rs:14`）把数据流方向定为 widget→model：将 tui_textarea 的 `cursor_position()` 返回的光标列号（col，字符索引）直接作为 `InputDocument::move_cursor()` 的字节位置使用。对 CJK 多字节字符（如"看"占 3 字节），字符索引 1 ≠ 字节位置 3，col=1 落在多字节字符中间被 `clamp_to_char_boundary` 修正到 0，导致下一个字符插入到现有字符之前（位置 0），造成顺序颠倒。

## 修复

随 feature #53 TUI Model/View 迁移结构性修复：

- 迁移把输入数据流**反向**为 model→widget，删除 `input_bridge.rs` 及 `mirror_input_area_to_model` 这条 textarea col→字节位置 的镜像路径。
- `InputModel::InputDocument`（原生按字节维护光标）成为唯一真源，通过 `input_adapter.rs::apply_input_changes_to_widget` 的 `input_area.set_text(整串文本)` 推给 widget。
- 不再用 textarea 列号反推字节位置，原根因结构性消失。
