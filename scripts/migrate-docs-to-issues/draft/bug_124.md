### #124 `/think` 命令输出不应渲染 Markdown

**状态**：待确认

**症状**：用户执行 `/think` 命令后，模型返回的思考内容在 TUI 输出区被当作 Markdown 渲染，导致思考文本中的 `*`、`_`、`` ` ``、`#` 等字符被错误解析为 Markdown 语法（如斜体、代码、标题），破坏纯文本思考过程的可读性。

**根因**：TUI 已将模型 thinking 流拆成独立 `ConversationBlock::Thinking` / `OutputBlockKind::ThinkingMessage`，但 `render_thinking()` 仍复用 `md::inline_markdown_lines()` 解析内容，导致 thinking 文本在专用 block 中仍被 Markdown 标记剥离并套用样式。

**修复**：`ThinkingMessage` 渲染改为本地纯文本按宽度换行：每行直接生成 THINKING 色 span，`plain` 与可见文本保持一致，不再经过 inline Markdown；保留空白过滤、宽度换行和 gutter 注入职责不变，普通 assistant Markdown 渲染不受影响。

**验证**：
- `cargo test -p cli test_thinking_renders_markdown_markers_as_plain_text -- --nocapture`
- `cargo test -p cli thinking -- --nocapture`
- `cargo test -p cli markdown`
- `cargo fmt --check`
- `cargo check -p cli`

**涉及路径**：
- `apps/cli/src/tui/render/output/blocks/thinking.rs`
