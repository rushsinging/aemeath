# Bug #62：Grep 工具执行中标题文字不可见但复制可见

| 字段 | 值 |
|------|-----|
| 优先级 | 中 |
| 发现日期 | 2026-05 |
| 归档日期 | 2026-05-30 |
| 状态 | 已确认修复 |
| 根因类别 | TUI 渲染 |

## 症状

TUI 中 Grep 工具运行态显示类似：

```text
● Grep /tui\.log/
  in /Users/guoyuqi/Nextcloud/work/claudecode/aemeath
```

屏幕上看不到 `Grep` 字样；选中复制时文本能正常复制出来，说明逻辑文本存在，仅渲染视觉不可见或颜色不可辨。

## 根因

Tool call 标题 span 的前景色与背景色过近，render 路径与 selection/copy 路径对样式合并不一致——后者读取的是 plain text，前者使用 styled spans，导致屏幕不可见但复制可见。

## 修复

随 #58 渲染管线重构结构性修复：

- ToolCall 标题在 `blocks/tool_call.rs` 用 `theme::TEXT` 前景渲染、图标用 `semantic_color`，与背景独立。
- 标题与图标颜色分离，避免运行态样式覆盖整个 span。
- 新增回归 `blocks/tool_call.rs::test_tool_call_title_visible_not_background_color`（fg ≠ bg 且 ≠ SURFACE）。
