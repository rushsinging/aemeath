# Bug #60：TUI 中 Markdown code 块复制时下划线被吞掉

| 字段 | 值 |
|------|-----|
| 优先级 | 中 |
| 发现日期 | 2026-05 |
| 归档日期 | 2026-05-23 |
| 状态 | 已确认修复 |

## 症状

TUI 输出区中 Markdown code 块里的文本包含下划线时，选中复制后的内容会丢失 `_`。例如 code 块中显示或期望复制 `CLAUDE_PROJECT_DIR`，实际粘贴结果变成 `CLAUDEPROJECTDIR`。

相关表现还包括标识符中的下划线曾被误判为 Markdown 斜体标记，例如 `CLAUDE_PROJECT_DIR` 被处理为 `CLAUDEPROJECTDIR`。

## 根因

复制/渲染路径中对 Markdown inline 标记的处理过于激进：

1. `_` 在标识符内部被误判为 emphasis/italic 标记。
2. `strip_inline_formatting` 和 `inline_markdown_spans` 没有充分区分 Markdown 语义标记与 code/标识符中的普通字符。
3. table 中 inline markdown（`code`、**bold** 等）也存在未正确渲染的问题，属于同一组 inline markdown 解析/渲染边界问题。

## 修复

1. 调整 `is_flanking` 检测：`_` 仅在前为空白/标点/行首、后为字母数字等满足 Markdown emphasis 条件时，才作为斜体标记处理。
2. 同步修复 `strip_inline_formatting`，避免删除标识符/code 文本中的普通下划线。
3. 同步修复 `inline_markdown_spans`，保证渲染路径与复制/纯文本提取路径一致。
4. 附带修复 table 中 inline markdown（`code`、**bold** 等）未渲染的问题。
