# Bug #26 几乎每次对话都触发 superpowers skill 调用

**状态**：已归档，不作为 Bug
**归档日期**：2026-05-06
**确认结果**：用户确认该项不算 Bug
**优先级**：中
**原根因类别**：SessionStart hook 注入提示过强或 skill 触发条件过宽

## 原现象

几乎每次对话开始时，LLM 都会主动通过 Skill 工具调用 superpowers 系列 skill（如 `superpowers:using-superpowers`、`superpowers:brainstorming` 等），即使用户的请求只是简单提问、查询信息或闲聊，并不需要任何 skill 介入。

## 原疑似原因

1. SessionStart hook（`~/.aemeath/hooks/superpowers-inject.sh`）提示文本偏强，容易让 LLM 把“调用 skill”当作默认动作。
2. `superpowers:using-superpowers` skill 的 description 包含 “Use when starting any conversation”，触发面较宽。
3. Skill 列表注入到 system prompt 后，LLM 倾向于优先尝试 Skill 工具。

## 归档原因

用户确认该项不算 Bug，不再作为活动 bug 跟踪。

## 后续处理

如后续仍希望降低不必要的 skill 调用频率，应作为体验优化或 feature 另行登记，而非 bug 修复。

## 涉及路径

- `~/.aemeath/hooks/superpowers-inject.sh`
- `aemeath-core/src/skill.rs`
- `aemeath-core/src/config/`
