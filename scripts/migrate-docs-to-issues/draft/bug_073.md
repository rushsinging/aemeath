# Bug #73：EnterWorktree 不能创建 worktree 导致 LLM 回退到主工作区 checkout

| 字段 | 值 |
|------|-----|
| 优先级 | 高 |
| 发现日期 | 2026-05 |
| 归档日期 | 2026-06-02 |
| 状态 | 已确认修复 |
| 修复 | 3bb1308, 29e3654 |

## 症状

用户要求“开个 wt”时，LLM 知道需要 worktree，但 `EnterWorktree` 只能进入已存在路径。目标不存在时模型可能回退到 Bash 手动执行 `git checkout -b` / `git worktree add`，容易把主工作区切到 feature 分支。

## 根因

`EnterWorktree` 工具语义只覆盖“进入已有 worktree”，没有覆盖自然语言里的“开 worktree”；工具描述也未明确禁止在主 checkout 中用 `git checkout -b` / `git switch -c` 代替 worktree。

## 修复

1. `EnterWorktree` 目标路径不存在时默认基于 `main` 创建 worktree 并进入。
2. `path` 改为可选，省略时从 `branch` 推导 `.worktrees/<安全分支名>`。
3. 推导 path 时仅保留安全字符，路径分隔符和敏感字符压缩替换为 `-`。
4. 工具描述和 schema 明确：开 worktree 必须调用 `EnterWorktree`，禁止在主 checkout 用 checkout/switch 代替。

## 验证

- 显式 path+branch 创建、branch 推导 path、敏感字符替换、缺少 path/branch 报错、嵌套进入拒绝和 schema 测试通过。
- 用户确认修复。
