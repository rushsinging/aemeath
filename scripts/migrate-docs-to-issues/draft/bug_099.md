# Bug #99：input area 里上下键始终翻看历史，无法上下移动光标

## 症状

在 input area 中按 Up/Down 键，始终触发历史翻看（`MoveHistoryPrevious/Next`），无法在多行输入中上下移动光标。

## 根因

`key.rs` 和 `key_event.rs` 将 Up/Down 硬编码映射为 `MoveHistoryPrevious/Next`，绕过了 `InputArea` 原有的 `move_up/move_down` 多行光标逻辑。

## 修复

新增 `InputIntent::MoveCursorUp/MoveCursorDown`，在 `InputModel` 中判断光标是否在首行/末行：在边界时翻历史，否则移动光标。`InputDocument` 新增 `move_up/move_down` 方法支持行间移动。

验证：633 tests passed。
