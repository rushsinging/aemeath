# Bug #90：TUI Edit 工具结果不渲染为 diff（只显示 ✓ Edit completed 摘要）

| 字段 | 值 |

| 根因类别 | TUI 渲染 / Edit 结果 DIFF 透传 |

## 症状

Edit 工具结果在 TUI 工具子块中只显示 `✓ Edit completed` 摘要，没有渲染为带行号、加减色的 diff 视图。

## 根因

`render_edit_diff`（解析 `---DIFF---` 出加减色 diff）只在 `render_tool_result` 里被调用，但 assembler 的 `tool_result_summary` 对 Edit 走默认 `format_result_summary` 返回 `✓ Edit completed`，使嵌入式 ToolResult 子块的 `result_text` 不含 `---DIFF---`，`render_edit_diff` 解析失败回退普通文本——`render_edit_diff` 在生产里从未拿到原文（单测喂原文故绿、生产坏）。

## 修复

与 #87 不冲突：#87 反对的是 orphan/非嵌入结果以原样文本泄漏到块外（仍走摘要抑制）；本修复仅对**嵌入式**且含 `---DIFF---` 的结果透传原文，由 `render_tool_result` 消费标记渲成 diff。

具体：`tool_result_summary` 在 `result.contains(DIFF_MARKER)` 时透传原文（`DIFF_MARKER` 提为 `pub(crate)` 复用，DRY）。

## 回归测试

`test_output_assembler_late_bound_tool_result_stays_inside_tool_block` 改为端到端断言子块渲染出 `- old`/`+ new` diff 行、无 `---DIFF---` 残留、无 `Edit completed`。
