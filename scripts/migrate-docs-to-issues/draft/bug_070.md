# Bug #70：TaskListCreate 第一次调用会超时

| 字段 | 值 |
|------|-----|
| 优先级 | 中 |
| 发现日期 | 2026-05 |
| 归档日期 | 2026-05-25 |
| 状态 | 已确认修复 |

## 症状

会话中首次调用 `TaskListCreate` 工具时出现超时。同一会话内随后再次调用表现正常，疑似仅首次触发。

## 根因

工具实现层（`TaskStore::create_list` + 锁顺序 + drop_archived_batch_tasks）纯内存操作，单测 `finished in 0.00s`，证伪「工具自身首次超时」假设。真正原因位于外层：LLM provider tool_use 投递阶段网络/握手延迟，被 dispatcher 120s timeout 错归到 TaskListCreate 工具名。

## 修复

1. `agent/runtime/src/agent.rs` 增加 `call_tool_with_timeout()`，统一包裹 `tool.call`，超时文案明确包含 `tool.call execution timed out`、`tool=<name>`、`timeout_secs=<n>`，与 LLM provider stream / tool_use 投递段区分。
2. `TaskListCreateTool::timeout_secs()` 从 120s 改为 5s。该工具是纯内存 O(1) 路径，若超过 5s 应快速暴露异常。
3. 新增回归测试：
   - `agent::tests::test_execute_tools_timeout_message_distinguishes_tool_call_execution`
   - `task_list_create::tests::test_task_list_create_timeout_is_short_for_memory_only_tool`
4. 工具层新增计时回归测试：
   - `test_task_list_create_first_call_is_fast`
   - `test_task_list_create_with_archived_batch_is_fast`

## 验证

2026-05-25 用户确认 bug #70 已修复。活动列表中移除 #70，并保留此归档记录。
