# Bug #50：input area 为空时按上键进入历史翻看模式

| 字段 | 值 |
|------|-----|
| 优先级 | 低 |
| 发现日期 | 2026-05 |
| 根因类别 | 逻辑错误 |

## 症状

input 为空时按上键无反应（不进入历史翻看）；期望行为是空 input 按上键进入历史翻看（类似 bash/zsh），非空 input 按上键只移动光标。

## 根因

`aemeath-cli/src/tui/input_area/editing.rs` 的 `move_up()` 方法在光标处于第 0 行时无条件调用 `history_up()`，不检查 input 是否为空。但实际期望行为是**空 input 进入历史，非空 input 不进入**。

初始修复方向反了（空 input 不进入、非空进入），后修正为正确方向。

## 修复

在 `move_up()` 的 `row == 0` 分支增加条件：`self.is_empty() || self.history_index.is_some()`。仅当 input 为空或已在历史浏览模式时才触发 `history_up()`。
