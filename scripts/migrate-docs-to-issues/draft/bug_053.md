# Bug #53：AskUserQuestion 选项未逐行显示，多个选项挤在一行

| 字段 | 值 |
|------|-----|
| 优先级 | 中 |
| 发现日期 | 2026-05 |
| 归档日期 | 2026-05-23 |
| 状态 | 已确认修复 |

## 症状

AskUserQuestion 展示选项时，多个选项没有逐行显示，而是挤在同一行，影响用户阅读和选择。

## 根因

1. AskUserQuestion 的 `options` 每个数组元素按一条 `OutputLine` 渲染。
2. 当模型把 A/B/C 等选项放在同一个字符串并用换行分隔时，换行符会被输出区 `sanitize_for_display` 当作控制字符移除，导致多个选项挤在一行。
3. 会话 `019e4bd2-31a3-72ce-a67a-b6bdf28fb5fd` 中还发现 LLM 将 A/B/C 选项直接塞进 `question`，原文无换行，说明工具描述对 `question` 与 `options` 的职责约束不够明确。

## 修复

1. 渲染 `options` 时按 `option.lines()` 拆行，确保单个 option 字符串内部的换行也能逐行展示。
2. 强化 AskUserQuestion 工具描述：
   - 多个选项必须逐项放入 `options` 数组；
   - `question` 只放问题正文；
   - 不应把 A/B/C 或多选项列表直接拼进 `question`。
3. 补充对应 schema 描述测试，约束工具描述不会回退。
