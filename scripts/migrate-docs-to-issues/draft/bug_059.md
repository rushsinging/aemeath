# Bug #59: Input area 翻历史记录时丢失换行且文本超出渲染框

- **发现日期**：2026-05
- **归档日期**：2026-05-22
- **状态**：已确认修复
- **优先级**：中

## 症状

1. 上下键翻历史记录时，多行历史输入中的换行被丢失，内容被压平成单行。
2. 历史浏览模式下，多行文本无法连续上下翻页（光标不在第一行时无法触发历史导航）。

## 根因

`editing.rs` 中 `set_text` 方法：

1. 使用 `text.lines()` 按 `\n` 分割后逐行 `insert_str`，但行间没有 `insert_newline()`，所有行被拼接在一起。
2. `delete_line_by_head` 只删除首行光标前内容，多行旧内容残留。
3. `move_up`/`move_down` 在历史浏览模式下仍检查光标行号（`row == 0` / `row >= line_count - 1`），多行文本恢复后光标不在首行，导致无法连续翻历史。

## 修复

提交 `b65ec5e`：

- `set_text` 改用 `split('\n')` + `insert_newline()` 在行间插入换行；清空改用 `select_all + cut`。
- `move_up`/`move_down`：在历史浏览模式下（`history_index.is_some()`）始终翻历史，不受光标位置限制。
- 5 个回归测试：多行保留、空文本、单行、历史上下翻。
