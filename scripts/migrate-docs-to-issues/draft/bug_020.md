# Bug #20 aemeath run --provider litellm 无反应直接退出

**状态**：✅ 已修复，用户已确认
**发现日期**：2026-04
**确认日期**：2026-04-29
**优先级**：高
**修复 commit**：待提交

## 症状

执行 `aemeath run --provider litellm` 时，进程无明显提示直接退出。TUI 模式下用户看不到错误原因，只能从日志文件中排查。

## 根因

`aemeath-cli/src/main.rs` 中多处 `process::exit(1)` 错误路径只写入 `log::error!`。在 TUI 模式下日志会路由到 `~/.aemeath/aemeath.log`，不会输出到 stderr，因此用户界面表现为“无反应直接退出”。

## 修复

在所有直接退出前增加 `eprintln!`，确保即使在 TUI 模式下，用户也能立即看到错误提示。

覆盖的错误路径：
1. provider 不识别
2. `models.default` 不匹配
3. API key 未设置
4. model 未配置
5. TUI 运行出错

## 涉及文件

- `aemeath-cli/src/main.rs`

用户已确认修复。
