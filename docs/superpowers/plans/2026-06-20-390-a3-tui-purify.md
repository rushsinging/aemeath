# #390 A3 — TUI 纯化（按 id 清占位 + 回显只认归宿事件 + MessagesSync 退出 display）Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** TUI 占位块携带 `InputId`、按 id 精确清除；用户回显**只**由 `UserMessagesAdded` 归宿事件驱动；`MessagesSync` 退出 display（降级为镜像 + 落盘）。

**Architecture:** A2 已让 runtime 回带 `UserMessagesAdded{items: Vec<AddedInput>}` 且 TUI 端 no-op。A3 把占位/回显从「MessagesSync diff 长度盲回显 + 全清占位」切换为「按 InputId 精确清占位 + 顺序回显」，MessagesSync 仅保留 `chat.messages` 镜像 + SaveSession。

**Tech Stack:** Rust，ratatui TUI，TEA-style `ConversationModel`（blocks + timeline 双投影）。

## Global Constraints
- **MUST** 回显只来自 `UserMessagesAdded`；占位只按 **InputId** 清。
- **MUST** `MessagesSync` 退出 display：删除其 clear 占位 + append 回显 + diff 逻辑；**保留** `self.chat.messages = msgs` 镜像更新 + `Effect::SaveSession`。
- **MUST** resume 显示不受影响：resume 走独立的 `render_history_message`（`apps/cli/src/tui/effect/session/resume.rs:23`），与 MessagesSync display 无关——**NEVER** 动 resume 路径。Task 4 须验证 `render_history_message` 仍完整。
- **MUST** 占位 InputId 与发往 runtime 的 `ChatInputEvent::UserMessage.id` 是**同一个**（submit 处生成一次，分别传入事件与占位）。
- **NEVER** 回退 #386：归宿事件 `items` 本就只含用户提交，天然不含 tool_result——比 `is_user_input` 更强；回显直接用 `item.text`。
- 验证门禁：`cargo clippy --all-targets --all-features`(0/0)、`cargo test --workspace`、`bash .agents/hooks/check-architecture-guards.sh`。cargo 前先 `source .cargo/set-target.sh`。

- [ ] **Step 3: 实现** —
```rust
            UiEvent::UserMessagesAdded(items) => {
                for item in items {
                    self.clear_queued_submission_echo_by_id(&item.id);
                    self.append_user_echo(item.text);
                }
                self.mark_output_dirty();
                return UpdateResult::one(Effect::SaveSession { notify: false });
            }
```
