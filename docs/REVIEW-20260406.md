## P0 — 严重（2 项）

### 1. API Key 明文泄露
**文件**: `env:4`  
**问题**: `MINIMAX_API_KEY=sk-cp-nO9qfL...` 被 git 跟踪，在 diff 中可见  
**风险**: Key 一旦推送到远程仓库将永久泄露  
**建议**: 
- 将 `env` 加入 `.gitignore`
- 使用 `git filter-branch` 或 BFG 清除历史
- 轮换该 key

### 2. Shell 注入检测误杀合法命令
**文件**: `aemeath-tools/src/bash.rs:check_shell_injection`  
**问题**: `check_shell_injection` 拦截所有 `&&`，包括 `cd /tmp && ls`、`cd dir && git log` 等安全命令。同时 `is_readonly_command` 也拦截 `&&`，导致双重拒绝  
**影响**: 模型常用的 `cd dir && command` 模式全部被阻断  
**建议**: 改为分析每个子命令的安全性，而非一刀切拒绝所有链式命令

## P1 — 高（3 项）

### 3. TUI 权限门控丢弃图片数据
**文件**: `aemeath-cli/src/tui/app.rs:925-935`  
**问题**: 权限检查后统一用 `results.into_iter().map(|(id, output, is_error, _)| ...)` 丢弃 images。即使 approved 工具返回了图片（如 Read 图片文件），也不传给模型  
**建议**: 恢复 `tool_results_rich` 路径，只对 denied 工具用简单结果

### 4. 并发工具结果顺序可能与调用顺序不一致
**文件**: `aemeath-core/src/agent.rs:execute_tools`  
**问题**: `join_all` 按注册顺序返回（正确），但 `results` 先 extend concurrent 再 append sequential。如果模型期望 tool results 按 tool_calls 原始顺序排列，可能导致 tool_use_id 与 tool_result 错位  
**建议**: 返回前按原始 tool_calls 顺序排序 results

### 5. 模糊匹配丢失 trailing newline
**文件**: `aemeath-tools/src/file_edit.rs:fuzzy_find_in_content`  
**问题**: `lines().join("\n")` 拼接不保留文件中的 trailing newline。`content.matches(&actual)` 可能匹配到错误位置  
**建议**: 匹配时考虑保留原始行尾，或在 `content.find(&actual)` 后用偏移量替换

## P2 — 中（3 项）

### 6. 配置合并无法区分"显式设置为默认值"
**文件**: `aemeath-core/src/config.rs:merge_config`  
**问题**: `overlay.api.provider != Provider::default()` 无法区分"未设置"和"显式设置为 Anthropic"  
**建议**: 用 `Option<Provider>` 区分

### 7. Scheduler 嵌套锁序风险
**文件**: `aemeath-core/src/scheduler.rs:complete_task`  
**问题**: 重试时持有 `active_tasks` 锁的同时获取 `task_queue` 锁  
**建议**: 文档化锁序，或先释放 `active_tasks` 再获取 `task_queue`

### 8. catch_unwind 后 UI 残缺无提示
**文件**: `aemeath-cli/src/tui/output_area.rs`  
**问题**: render panic 被捕获后静默继续，用户看到半渲染界面  
**建议**: 在 status bar 显示 "Render error, try resizing"
