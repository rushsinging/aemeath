## 架构总览

| Crate | 职责 | 代码量 | 评价 |
|-------|------|--------|------|
| `aemeath-core` | Agent/Tool/Message 抽象、配置、权限、MCP、任务调度 | ~5,000 行 | 设计合理 |
| `aemeath-llm` | LLM Provider 抽象、流式传输、Anthropic/OpenAI 兼容 | ~1,500 行 | 质量最高 |
| `aemeath-tools` | 21+ 个工具实现（文件/搜索/Bash/Web/Task 等） | ~3,500 行 | 安全问题已修复 |
| `aemeath-cli` | CLI 入口、TUI（ratatui）、REPL、渲染系统 | ~9,000 行 | 功能完整 |

**架构优点**:
- Workspace 分层清晰，crate 依赖方向单一 (cli → core/llm/tools)
- 异步架构一致使用 Tokio，工具系统通过 `#[async_trait]` 统一接口
- 配置系统支持 5 级层叠覆盖（默认→全局→项目→环境变量→CLI 参数）
- 支持 9 个 LLM provider (Anthropic, OpenAI, OpenRouter, DeepSeek, Moonshot, Zhipu, DashScope, MiniMax, OpenAI-Compatible)
- SSE 流式解析含 90s 空闲超时 + 30s 停滞检测 + CancellationToken

## P0 — 严重问题（全部已修复）

### 1. Bash 分号注入绕过安全检查 — ✅ 已修复
**位置**: `aemeath-tools/src/bash.rs`  
**问题**: 安全检查只看管道第一个命令，`ls; git reset --hard` 可绕过  
**修复**: 添加 `check_shell_injection()` 函数，检查 `;`、`&&`、`||`、`$(...)`、反引号等 shell 注入模式  
**状态**: ✅ 已修复

### 2. Semaphore Permit 立即释放 — ✅ 已修复
**位置**: `aemeath-core/src/agent.rs:63`  
**问题**: `_permit` 在 await 后被 drop，并发限制完全失效  
**修复**: `_permit` 的 scope 现在覆盖整个 `tool.call` + `timeout` 块，执行完才释放  
**状态**: ✅ 已修复

### 3. TaskStore 潜在死锁 — ✅ 已修复
**位置**: `aemeath-core/src/task.rs:121-130`  
**问题**: `is_blocked()` 在持有锁时调用 `store.get()` 再次获取 `tokio::sync::Mutex`  
**修复**: 改为先 snapshot 再检查，不嵌套锁定。添加了锁序文档注释  
**状态**: ✅ 已修复

## P1 — 重要问题

### 4. 消息历史克隆而非共享 — ✅ 已修复
**位置**: `aemeath-cli/src/tui/app.rs`  
**问题**: TUI 每次 spawn 后台任务时深克隆 `Vec<Message>`，助手回复不回传主线程  
**修复**: 添加 `MessagesSync` 事件，后台任务完成前将完整消息历史回传主线程  
**状态**: ✅ 已修复

### 5. ~~state.rs / session.rs 重复 Session 定义~~ — ✅ 已修复
**位置**: `aemeath-core/src/state.rs:137` + `aemeath-core/src/session.rs:24`
**问题**: 两套不同消息结构的 Session 类型，API 表面混乱
**修复**: 将 state.rs 的 Session 重命名为 InternalSession，添加 #[deprecated] 别名保持兼容性
**状态**: ✅ 已修复

### 6. 文件路径穿越 — ✅ 已修复
**位置**: `aemeath-tools/src/file_read.rs`  
**问题**: FileRead 无路径归一化，`../../../etc/passwd` 可读取项目外文件  
**修复**: 添加 `validate_and_normalize_path` 路径归一化 + `has_traversal_attempt` 遍历检测，限制在 workspace 边界内  
**状态**: ✅ 已修复

### 7. MCP 工具盲代理 — ✅ 已修复
**位置**: `aemeath-core/src/mcp_manager.rs:360`  
**问题**: 无参数 schema 校验，完全信任 MCP 服务端返回的工具定义  
**修复**: 添加 `validate_mcp_input()` 函数进行输入校验  
**状态**: ✅ 已修复

### 8. Streaming 失败静默降级 — ✅ 已修复
**位置**: `aemeath-llm/src/providers/openai_compatible.rs`  
**问题**: 流式传输失败自动切换非流式模式，用户完全无感知  
**修复**: 降级时输出 `[warn] Streaming failed, falling back to non-streaming: {error}` 到 stderr  
**状态**: ✅ 已修复

### 9. 序列化静默失败 — ✅ 已修复
**位置**: `aemeath-llm/src/providers/openai_compatible.rs:120-121`  
**问题**: `serde_json::to_string().unwrap_or_default()` 失败返回空字符串  
**修复**: 改为 `.map_err()` 传播错误，返回 `LlmError::Config`  
**状态**: ✅ 已修复

### 10. 持久化无 Schema Version — ✅ 已修复
**位置**: `aemeath-core/src/task.rs`  
**问题**: 无版本号，schema 变更时无法向后兼容  
**修复**: `TaskStoreData` 添加 `version: u32` 字段 + `default_version()` 默认值  
**状态**: ✅ 已修复

### 11. 5xx 状态码不重试 — ✅ 已修复
**位置**: `aemeath-llm/src/providers/openai_compatible.rs`  
**问题**: 只重试 429，502/503/504 直接报错  
**修复**: 添加 `status >= 500 && < 600` 重试逻辑，输出 `[warn] Server error {status}, will retry`  
**状态**: ✅ 已修复
